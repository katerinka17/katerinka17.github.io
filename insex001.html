Головною особливістю Веб 3.0 буде домінування веб-застосунків, при чому навіть над звичайними програмами для ОС. Тенденція перенесення центру уваги з ПК користувача (клієнта) на Інтернет-ресурси (сервери) простежується на багатьох продуктах і відповідає ідеології «хмарних обчислень».Разом сюди відносяться web- OC, хмаркові OC,універсальні OC для смартфонів типу OC «Android» тощо.

Чому створена Chrome OS?

Тому,що Google має свою ідеологію розвитку інтернету або Веб 3.0, це повний перехід на хмаркові технології. З другого боку,Google універсальні браузерні рушії WebKit3 (2010) він був переписаний інженерами Google з нуля, та Blink(2013). В свою чергу в основі браузерних рушіїв лежать технології JavaScriptCope та V8.

V8-це власна розробка Google для допомоги виходу JavaScript на рівень повноцінної мови. Його особливістю є компіляція сирцевого коду JavaScript,безпосередньо у власний машинний код,минаючи стадію проміжного байт-коду. Також ефективна система управління пам’яттю,яка ведедо зменшення пауз на прибирання сміття, V8 зупиняє виконання коду під час виконання прибирання сміття,введення вбудованих кешів.

Chrome OS позиціонується як операційна система для різних пристроїв

Chrome OS позиціонується як операційна система для різних пристроїв — від маленьких нетбуків до повнорозмірних настільних систем, наразі система може працювати тільки на нетбуках. Дизайн OC витриманий в мінімалістському стилі в дусі браузера Google Chrome.Не виключений варіант об’єднання двох операційних систем компанії Android та Chrome OS, в єдиний продукт, злиття споживачів, які хотіли б, крім веб-застосунків, запускати на одному і тому ж комп’ютері ще програми для Android. Всі існуючі веб-застосунки працюють на OC Google Chrome автоматично, а нові можна писати з використанням будь- якої веб-технології. Інтерфейс Chrome OC ідентичний браузеру Chrome,доповненою розташованою зліва вкладкою з переліком застосунків і спливаючими панелями, для швидкого відображення таких міні-застосунків, як блокнот, музичний і відеопрогравач, клієнт обміну миттєвими повідомленнями тощо.

Браузер Chrome не можна замінити на альтернативний, наприклад Firefox, він є компонентом OC. Так само при видаленні браузера IE y Win 95-98 OC переставала працювати.Стандартні програми для OC є веб-застосунками виробництва Google,але можна запустити іншу веб-програму , наприклад флеш-гру або Office 365. Прямого доступу до системних даних немає,Chrome OC оновлюється автоматично. Під час завантаження OC кожен її компонент криптографічний, перевіряється на цілісність, що виключає проникнення шкідливого коду. У разі руйнування системи вона автоматично відновлюється з образу.Chrome OC закрита від доступу і не довіряє жодному застосунку, змушуючи їх виконувати в «пісочниці » браузера. Всі призначені для користувача дані шифруються, при втраті комп’ютера прочитати їх буде неможливо. Завдяки автоматичній синхронізації локальної інформації та даних Google «хмарі»користувач завжди має до найостаннішої їх версії, працюючи на іншому комп’ютері. У випадку Chrome OC гіпотезу професора Таненбаума можна вважати підтвердженою, єдиний її недолік — це відсутність колективної роботи мережі, тобто «розподілених хмарних обчислень». Аналогічно до Chrome OC , була створена Firefox OC-це операційна система, ошолошена організацією Mozilla, у липні 2012. Операційна система заснована на браузерному рушії Gecko, та цілком спирається на відкриті веб-стандарти, надаючи розробникам застосунків засновані на HTML 5 компоненти для задіяння всіх можливостей апаратних пристроїв. На створення проекту розробників підштовхнула поява рушія для обробки PDF засобами HTML 5 та JavaScript.

У випадку Firefox OC гіпотезу Таненбаума можна вважати підтвердженою- неуспіх цієї технології пов’язаний з тим,що вона використовує рушій, а не віртуальну машину, недостатньо абстрагована від апаратної платформи, орієнтована на використання для смартфонів. Обмін великими масивами даних сервер-клієнт ведеться за допомогою JSON, тобто використовуються noSQL-технології.

Хмарні технології-це програмно-апаратне забезпечення, доступне користувачеві через Інтернет або локальну мережу у вигляді сервісу,що дозволяє використовувати зручний інтерфейс для віддаленого доступу до виділених ресурсів (обчислювальних ресурсів, програм і даних ). Комп’ютер користувача виступає при цьому рядовим терміналом, підключеним до Мережі. Комп’ютери, які здійснюють « cloud computing», називаються «обчислювальною хмарою ». Але навантаження між комп’ютерами ,що входять в «обчислювальну хмару», не розподіляється рівномірно, основні його частини лежать на серверах. Принцин хмарних обчислень- комп’ютер користувача розглядається лише як частина обчислювальної системи,безпосередньо взаємодіє з користувачем, у той час, як більша частина ресурсів надається серверами в мережі, власне вони і є основою компонента хмари, а не клієнта. Користувач отримує доступ до своїх файлів з будь-якого підключеного до Інтернету комп’ютера. У хмарі моду зберігатися не тільки дані, але й додатки. Практично включається можливість використання контрафактного софту, стає зручно продавати програми : вони можуть взагалі не передаватись користувачеві. Полегшується технічна підтримка- проблеми на стороні клієнта знімаються очищенням кешу браузера чи перезагрузкою клієнта.

Для роботи хмарних додатків потрібні великі обчислювальні потужності, які повинні бути забезпечені постачальникам ПЗ та ПК. На домашніх ПК хмарні OC цілком підходять на роль другої OC, яку можна швидко завантажети тільки для виходу в Інтернет. WebOC — мають на увазі рішення DaaS, що працюють в будь-якому звичайному web-браузері і включають базовий набір ПЗ.

Серверна частина WebOC може розміщуватися на звичайному хостінгу або будь-якому комп’ютері з встановленим web-сервером. Вікна додатків, що входять до складу такої OC, зазвичай відкриваються на тій же web-сторінці, навіть немає необхідності перемикатися між вкладками браузера.

Що таке Веб 3.0?

Web 3.0 - це програмна система, призначена для підтримки взаємодії типу комп'ютер-комп'ютер через Інтернет. Web 3.0 - це нова технологія створення та обслуговування інтернет-додатків. Термін - несталий, в англомовному середовищі більш популярно назва Semantic Web (семантична павутина).

Що таке Web 3.0

Web 1.0

Власник сайту є джерелом інформації, читач є її споживачем. Софт, який це забезпечує, називається Веб 1.0.

Web 2.0

Власник сайту керує інформацією, яку постачають користувачі згідно з правилами власника.

Web 3.0

Вищим рівнем Вебу є такий стан, коли користувачі сайту є його власниками і самі наповнюють його інформацією згідно з власними правилами.

Стандартна реалізація Python, так звана СPython (реалізація на ANSI C ). C — це еталонна реалізція, вона як правило працює швидше, стійкіше та краще, ніж альтернативні реалізації.

JPython , основна мета — тісна інтеграція з мовою програмування Java. Реалізація Jython складається з Java-класів, які виконують компіляцію програмного коду на мові Python в байт-код Java і потім передають отриманий байт код віртуальній машині Java ( JVM ).

Мета Jython полягає в тому, щоб дозволити програмам на мові Pyton управляти Java — додатками, СPython може керувати компонентами на мовах С / С++. Оскільки програмний код на Python трансюється в байт-код Java, під час виконання він веде себе так само, як справжня програма на мові Java. Програми на Jython можуть виступати в якості аплетів і сервлетів , створювати графічний інтерфейс з використанням механізмів Java тощо. Більш того, Jython забезпечує підтримку можливості імпортувати і використовувати Java- класи в програмному коді Python.Реалізація Jython забезпечує більш низьку швидкість виконання і менш стійка в порівнянні з СPython.

IronPython — це реалізація призначена для забезпечення інтеграції програм Python з додатками , створеними для роботи в середовищі Microsoft . NET Framework операційної системи Windows , а також в Mono— відкритому розділі . NET Framework ( еквівалентні ) для Linux. IronPython дозволяє програмам на основі Pyton грати роль як клієнтських , так і серверних компонентів . Розробка ведеться компанією Microsoft..

Існують інші реалізації Python, включаючи динамічний компілятор Psyco і транслятор Shedskin C++, які намагаються оптимізувати основну модель виконання. Система Psyco -це компонент, який розширює модель виконання байт — коду, що дозволяє програмам виконуватися швидше. Psyco є розширенням PVM, яке збирає і використовує інформацію про типи, щоб транслювати частини байт-коду програми в істинний двійковий машинний код, який виконується набагато швидше. Для такої трансляції не потрібно вносити зміни до початкового коду або виробляти додаткову компіляцію в ході розробки.

Під час виконання програми, Psyco збирає інфорамцію про типи об’єктів, і потім цю інформацію використовує для генерації високоефективного машинного коду, оптимізованого для об’єктів цього типу. Після цього вироблений машинний код замінює відповідні ділянки байт-коду, тим самим збільшується швидкість виконання. В ідеалі деякі ділянки програмного коду під управління Psyco можуть виконуватися так само швидко, як скомпільований код на мові С .

  • Psyco забезпечує збільшення швидкості від 2 до 10 разів, але він в даний час здатний генерувати машинний код тільки для архітектури Intel x86.
  • Проект РуРу, який являє собою спробу переписати PVM з метою оптимізації коду.
  • Shedskin — це система , яка перетворює вихідний код на мові Python в вихідний код на мові С++ , який потім може бути скомпільовано в машинний код. Крім того, система реалізує платформонезалежний підхід до виконання програмного коду Python.
  • Іноді необхідно зі своїх програм на Python створювати самостійні виконувані файли. Це необхідно швидше для упаковки і розпосюдження програм .
  • Фіксовані виконавчі файли об’єднують в єдиний файл пакета байт-код програм, PVM і файли підтримки, необхідні програми. В результаті виходить єдиний виконуваний файл, наприклад файл з розширенням «.exe» для Windows.
  • Фіксовані виконавчі файли мають чималий розмір, бо містять в собі PVM, але за сучасними мірками не можна назвати незвичайно великими. Так як інтерпретатор Python вбудований безпосередньо в фіксовані виконавчі файли, його установка не є обов’язковою вимогою для запуску програм.
